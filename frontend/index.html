📋 Key Capabilities & Use Cases

GeoKG 기반 디지털 트윈 시스템의 핵심 기능과 차별점입니다.

💥 연쇄 영향도 분석 (Cascade Impact Analysis) ▼

한 entity 변경 시 N-hop 관계를 따라 영향받는 모든 entity를 자동으로 추적합니다.

예시: 6th Ave 수도관 파열 →
Road(6th Ave) → ALONG → Facility(수도관)
  → ADJACENT_TO → Building(BoA Tower)
  → MONITORS ← Sensor(온도)
그래프 탐색 한 번으로 연쇄 영향 즉시 파악

🔍 KG Analysis → Impact Analysis 탭에서 실시간 확인 가능

정적 관계 모델에서 벗어나, 동적 이벤트 전파와 실시간 N-hop 영향도 시각화를 구현. 기존 SQL JOIN으로 복잡했던 연쇄 분석을 그래프 패턴 매칭으로 단순화.
🌐 GSID 공간 식별 체계 (Spatial Identification) ▼

S2 Cell 기반 글로벌 고유 식별자(GSID v7.0)로 모든 entity를 공간적으로 인코딩합니다.

형식: BL-89C25901-G0-00-000001-5
도메인(2) - S2Cell(8) - 고도(2) - 세대(2) - 일련번호(6) - 체크섬(1)

도메인 매핑: BL=Building, RC=Road, OM=Vehicle, OF=Sensor/Camera, NT=Tree, ZU=Zone

도시 간 데이터 통합 시 매핑 테이블이 불필요하며, Luhn mod-36 체크섬으로 데이터 무결성을 보장합니다.

독자적 GSID 체계를 설계. S2 Geometry 기반 공간 파티셔닝, 체크섬 검증, 계층적 SubType 분류까지 포함하여 글로벌 스케일 데이터 통합 가능.
⚡ 실시간 상태 추적 (Real-time State Tracking) ▼

WebSocket 기반으로 센서, 차량, 주차장 상태를 실시간 업데이트합니다.

동작 흐름:
센서값 변경 → WebSocket 이벤트 → KG State Domain(SO) 업데이트
→ 3D 뷰 시각화 반영 + 대시보드 수치 갱신 (동시)

State Domain(SO)이 시간적 변화 이력을 관리하여, 과거 상태 추적 및 트렌드 분석이 가능합니다.

기존 배치 기반 정적 KG 업데이트 방식과 달리, WebSocket 스트리밍으로 실시간 Digital Twin을 구현. 센서→KG→시각화 파이프라인이 자동으로 동기화.
🔎 공간-시맨틱 통합 쿼리 (Spatial-Semantic Query) ▼

Cypher 기반 그래프 패턴 매칭으로 공간과 의미를 동시에 쿼리합니다.

예시: "42nd St에 있는 bus 근처 카메라"
MATCH (v:Vehicle)-[:ON_ROAD]->(r:Road), (c:Camera)-[:ALONG]->(r) WHERE r.name='West 42nd Street' RETURN c, v

SQL: 3-4개 테이블 JOIN 필요 → Cypher: 패턴 매칭 한 줄

🔍 KG Analysis → Cypher 탭에서 직접 실행 가능

이론적 쿼리 모델을 Neo4j Cypher 엔진으로 구현. GSID 기반 검색과 결합하여 공간+시맨틱+식별자 3중 쿼리 지원.
🏗️ 3-Domain 온톨로지 (Three-Domain Ontology) ▼

Entity(EO), State(SO), Change(CO) 3단계 도메인 모델로 도시 장면을 체계적으로 표현합니다.

Entity Domain (EO): Building, Road, Vehicle, ParkingLot, ParkingSpace, Sensor, Camera, Tree, Facility, Zone — 10개 타입

State Domain (SO): TrafficState, EnvironmentState, OccupancyState, EquipmentState, WeatherState — 5개 타입

Change Domain (CO): PositionChange, StatusChange, EnvironmentChange, MaintenanceEvent — 4개 타입

관계: CONTAINS, ADJACENT_TO, CONNECTED_TO, NEAR, ON_ROAD, ALONG, MONITORS, SERVES, HAS_STATE — 9개 관계 타입
개념적 온톨로지(Ont = <CO, EO, SO>)를 Neo4j 그래프 DB에 완전히 구현. 제약조건, 인덱스, GSID 통합까지 포함한 Production-ready 스키마.
